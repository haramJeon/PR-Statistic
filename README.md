# PR Statistic

GitHub PR 리뷰어 통계 분석 도구. 로컬 웹 서버를 띄워 브라우저에서 저장소를 추가하고 팀원별 리뷰 활동을 분석합니다.

## 실행

```bash
python pr_stats.py
```

브라우저가 자동으로 열립니다.

### 옵션

| 옵션 | 기본값 | 설명 |
|------|--------|------|
| `--token` | (자동 감지) | GitHub Personal Access Token |
| `--state` | `closed` | PR 상태 (`open` / `closed` / `all`) |
| `--max-prs` | `200` | 저장소당 최대 수집 PR 수 |
| `--port` | (랜덤) | 로컬 서버 포트 |

```bash
python pr_stats.py --port 8080 --state all --max-prs 500
```

### 토큰 자동 감지 순서

1. `--token` 인자
2. 환경변수 `GITHUB_TOKEN`
3. `gh auth token` (GitHub CLI)
4. `git credential`

## 사용 방법

1. 좌측 사이드바에서 저장소 추가 (`https://github.com/owner/repo.git` 또는 `owner/repo`)
2. 날짜 범위 설정 (기본값: 최근 2주)
3. 분석할 팀원 선택
4. **분석 시작** 클릭
5. 분석 완료 후 **결과 저장 (HTML)** 으로 리포트 다운로드

## 제공 차트

| 차트 | 설명 |
|------|------|
| 리뷰 참여 수 vs PR 생성 수 | 팀원별 리뷰 참여 및 PR 생성 현황 |
| 참여율 | 리뷰 요청 대비 참여 비율 (본인 PR 포함) |
| 응답 / 승인 시간 | 평균 응답시간 및 승인시간 비교 |
| 첫 리뷰 응답시간 분포 | 히스토그램 |
| 머지 소요시간 | 히스토그램 |
| 리뷰 후 머지까지 | 히스토그램 |
| 월별 머지 PR | 추세선 |
| 코드 변경량 vs 머지 소요시간 | 산점도 (hover 인터랙션) |
| 리뷰어 산점도 | 응답시간 vs 참여율, 4분면 색상 구분 |

## 주요 계산 방식

- **응답시간**: 주말(토·일) 제외 영업시간 기준
- **참여율**: `(리뷰 참여 수 + 본인 PR 수) / (리뷰 요청 수 + 본인 PR 수) × 100`
- **최다 리뷰어**: 리뷰 참여 수 + 본인 PR 수 합산 기준

## 결과 저장

분석 완료 후 **결과 저장 (HTML)** 버튼으로 단일 HTML 파일 다운로드.

- 정적 차트는 SVG 인라인 삽입 (외부 의존 없음)
- 코드 변경량 vs 머지 소요시간 차트는 hover 인터랙션 유지 (Plotly CDN 필요)
- 파일명: `pr-stats_YYYYMMDD_YYYYMMDD.html`

## 요구사항

- Python 3.7+
- 표준 라이브러리만 사용 (별도 패키지 설치 불필요)
- GitHub API 접근 가능한 토큰
